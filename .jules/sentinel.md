## 2024-05-06 - Silent Failure Anti-Pattern & RBAC Integer Edge Case in Gin Controllers
**Vulnerability:** A `DeleteUser` function had two critical issues: 1) A silent failure where it returned `{"success": true}` when `model.HardDeleteUserById` failed with an error. 2) The RBAC integer comparison `if myRole <= originUser.Role` lacked an exception for root users (`common.RoleRootUser`), meaning a root user couldn't delete other root users.
**Learning:** Returning `{"success": true}` inside an `err != nil` block is a severe anti-pattern in this Gin codebase, leading to false positives on deletion failures and potential data inconsistency or bypasses. Furthermore, since roles are integers where higher numbers mean *lower* privileges, missing `&& myRole != common.RoleRootUser` locks out supreme administrators.
**Prevention:** Always use `common.ApiError(c, err)` on error branches in controllers. When comparing roles (e.g. `user.Role >= myRole`), ensure `common.RoleRootUser` is explicitly exempted to avoid restricting root administrators.

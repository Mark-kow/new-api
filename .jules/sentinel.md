## 2024-04-28 - Silent Failure Anti-Pattern in Gin Controller Responses
**Vulnerability:** The `DeleteUser` function in `controller/user.go` had a silent failure vulnerability where an error in database deletion (`model.HardDeleteUserById(id)`) was caught, but the response mistakenly declared `{"success": true, "message": ""}` instead of returning an error using `common.ApiError(c, err)`. Furthermore, a successful deletion lacked any JSON response payload.
**Learning:** Returning success payloads on `err != nil` conditions tricks administrators and clients into believing malicious or targeted accounts were successfully deleted when they were not. This is a recurring pattern in the Gin controllers where error paths are accidentally given success response bodies.
**Prevention:** Always verify that error handling paths return standard error responses (e.g., `common.ApiError(c, err)`) and that successful paths explicitly send the expected success JSON payload. Do not copy-paste `c.JSON(..., gin.H{"success": true})` into `if err != nil` blocks.
